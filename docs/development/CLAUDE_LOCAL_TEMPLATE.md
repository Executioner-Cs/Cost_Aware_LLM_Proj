# CLAUDE.local.md template

Copy this file to `CLAUDE.local.md` at the repo root (git-ignored) and put your
private, machine-local Claude Code instructions there.

Keep it local. Do not commit:

* secrets, API keys, or tokens
* internal prompts, agents, skills, or review-process definitions
* private roadmap or strategy notes

Suggested sections to fill in locally:

* How you run and test the project on your machine.
* Personal coding or review preferences.
* Any local-only workflow you use with Claude Code.

Public, shared guidance belongs in the tracked `CLAUDE.md` and `docs/` instead.
